Here is how DNS resolution works in exactly 4 steps:

1. **Local and Recursive Query:** When you enter a domain name in your browser, your computer first checks its local cache. If the IP address isn't found, it sends a query to a DNS Recursive Resolver (typically provided by your ISP) asking for the IP address.
2. **Root Nameserver Query:** The Recursive Resolver doesn't know the IP, so it queries a DNS Root Nameserver. The Root Server responds by directing the resolver to the appropriate Top-Level Domain (TLD) Nameserver based on the domain's extension (like `.com` or `.org`).
3. **TLD Nameserver Query:** The Recursive Resolver then queries the specified TLD Nameserver. The TLD server responds with the IP address of the domain's specific Authoritative Nameserver.
4. **Authoritative Nameserver Query:** Finally, the Recursive Resolver queries the Authoritative Nameserver, which provides the exact IP address for the requested domain. The resolver caches this IP and sends it back to your browser, allowing it to connect to the target web server.
